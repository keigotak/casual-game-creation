# Translate - 翻訳実行

あなたはLocalization Agentとして、ゲームの翻訳を実行します。

## 対象

$ARGUMENTS（言語コードまたは「all」を指定）

例:
- `/translate en` - 英語翻訳を実行
- `/translate ja` - 日本語翻訳を実行
- `/translate all` - 全言語を更新

## 手順

### 1. 現状確認

以下を確認してください：
- `src/i18n/locales/` 内の言語ファイル
- `/i18n-check` の最新レポート（あれば）
- ソースコード内の `t()` 呼び出し

### 2. 翻訳キーの抽出

ソースコードから全ての `t('...')` パターンを抽出し、
必要な翻訳キーのリストを作成してください。

```javascript
// 検索パターン
t('key')
t('key', { param: value })
t(`dynamic.${variable}`)
```

### 3. 翻訳実行

#### 日本語 → 英語
- 自然な英語表現を使用
- ゲームUIに適した簡潔さを維持
- 文化的な差異を考慮

#### 英語 → 日本語
- 自然な日本語表現を使用
- カタカナ語の適切な使用
- 敬語/カジュアルの一貫性

### 4. 翻訳ファイル更新

`src/i18n/locales/{lang}.json` を更新してください。

```json
{
  "namespace": {
    "key": "翻訳テキスト",
    "key_with_param": "{{param}}を含むテキスト"
  }
}
```

### 5. 検証

- [ ] JSON構文が正しいか
- [ ] 全キーが存在するか
- [ ] 補間パラメータが一致するか
- [ ] 文字化けがないか

### 6. レポート

```markdown
## 翻訳完了レポート

**日時**: YYYY-MM-DD HH:MM
**対象言語**: [言語]

### 統計
- 新規追加: X件
- 更新: X件
- 削除: X件

### 追加されたキー
| キー | 値 |
|-----|-----|
| xxx | xxx |

### 注意事項
[翻訳時の判断、確認が必要な点など]
```

## 翻訳品質ガイドライン

### ゲームUI翻訳の原則
1. **簡潔さ**: ボタンは2-4文字、ラベルは短く
2. **一貫性**: 同じ概念には同じ言葉
3. **プレイヤー視点**: 「あなたは」より「～しよう」

### 例

| 日本語 | 英語 |
|--------|------|
| スタート | Start |
| もう一度 | Retry |
| タイトルへ | Title |
| 設定 | Settings |
| スコア | Score |
| ハイスコア | High Score |
| ゲームオーバー | Game Over |
| クリア | Clear! |

## 次のステップ

翻訳完了後：
1. `/i18n-check` で翻訳漏れがないか再確認
2. 問題なければ `/asset-design` → `/sound-design` へ
3. 全準備完了後は `/release` でリリース
