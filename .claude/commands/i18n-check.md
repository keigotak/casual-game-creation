# i18n Check - 国際化チェック

あなたはLocalization Agentとして、Webゲームの国際化状況をチェックします。

## チェック対象

$ARGUMENTS（指定がない場合は全体チェック）

## チェック手順

### 1. i18n設定の確認

以下のファイルを確認してください：
- `src/i18n/index.js` - i18n設定
- `src/i18n/locales/*.json` - 翻訳ファイル
- `package.json` - i18next関連の依存関係

### 2. ハードコード文字列の検出

以下のパターンでハードコードされた文字列を検索してください：

```javascript
// 検出すべきパターン
- JSX内のテキスト: <div>日本語テキスト</div>
- テンプレートリテラル内のテキスト: `${score}点`
- 文字列リテラル: "ゲームオーバー"
- alert/confirm内: alert("エラー")

// 除外すべきパターン
- console.log内のデバッグ用テキスト
- コメント
- CSSクラス名
- 変数名
```

特に以下のファイルを重点的にチェック：
- `src/*.jsx`
- `src/components/*.jsx`
- `src/game/*.js`

### 3. 翻訳キーの整合性チェック

全ての言語ファイル間でキーを比較し、以下を検出：

1. **欠落キー**: ある言語にあって他にないキー
2. **未翻訳**: 値が空または元言語のまま
3. **重複キー**: 同じキーが複数定義
4. **未使用キー**: コードで使われていないキー

### 4. t() 関数の使用確認

以下をチェック：
- `t('key')` の呼び出しが正しい形式か
- 動的キーの使用: `t(\`prefix.${variable}\`)` の妥当性
- 補間パラメータ: `t('key', { value })` の正確性

### 5. レポート出力


## 出力先

レポートは `project_memory/translations/` に保存してください。
ファイル名: `YYYY-MM-DD-i18n-check.md`

### 次のステップ
1. ハードコード文字列があれば `/implement-feature` で翻訳キーに置換
2. 欠落・未翻訳キーがあれば `/translate [lang]` で翻訳実行
3. 問題なければ `/asset-design` → `/sound-design` → `/write-description` → `/release` へ
