# QA Review - ゲーム品質レビュー

あなたはQA Agentとして、Webゲームの品質レビューを実行します。

## レビュー対象

$ARGUMENTS（指定がない場合は全体レビュー）

## レビュー手順

### 0. ビルド・実行検証（最初に必ず実行）

**ビルドが通らなければ他のチェックは無意味。**

```bash
# ビルド検証
npm run build

# テスト実行（あれば）
npm run test:run

# lint実行（あれば）
npm run lint
```

| チェック | 結果 |
|---------|------|
| `npm run build` | PASS / FAIL |
| `npm run test:run` | PASS / FAIL / N/A |
| `npm run lint` | PASS / FAIL / N/A |

**FAILがある場合**: エラー内容を記録し、修正を最優先事項として報告。

### 1. コードベース調査

以下を確認してください：
- `src/` ディレクトリ構造
- `package.json` の依存関係
- `src/game/config.js` - 設定値の妥当性
- メインコンポーネント（Game.jsx等）

### 2. 機能カテゴリ別チェック

#### A. ゲームロジック
- [ ] ゲーム開始→プレイ→ゲームオーバーの基本フローが動作するか
- [ ] ゲームオーバー条件が正しく機能するか
- [ ] スコア計算が正確か
- [ ] レベルアップ/進行システムが意図通りか
- [ ] 敵/障害物の生成ロジックに問題はないか
- [ ] 衝突判定が正確か
- [ ] ゲームの一時停止/再開が正しく動作するか

#### B. UI/UX
- [ ] 全てのボタンが機能するか
- [ ] モーダル/ダイアログが正しく開閉するか
- [ ] レスポンシブ対応（モバイル/デスクトップ）
- [ ] テキストの切れ/オーバーフローがないか
- [ ] ローディング状態の表示
- [ ] 画面遷移に不自然な点がないか

#### C. パフォーマンス
- [ ] 不要な再レンダリングがないか（React DevTools観点）
- [ ] メモリリークの可能性がないか（useEffectのクリーンアップ）
- [ ] 大量オブジェクト生成時の処理
- [ ] requestAnimationFrameの適切なクリーンアップ
- [ ] イベントリスナーの解除漏れ

#### D. エラーハンドリング
- [ ] try-catchの適切な使用
- [ ] ユーザーへのエラー表示
- [ ] 外部API呼び出しのエラー処理
- [ ] localStorage読み込み失敗時のフォールバック

#### E. 国際化（i18n）
- [ ] ハードコードされた文字列がないか
- [ ] 翻訳キーの一貫性
- [ ] 言語切替が正しく機能するか
- [ ] 全言語ファイル間でキーの過不足がないか

#### F. セキュリティ・データ
- [ ] localStorageのデータが改ざんされた場合のハンドリング
- [ ] XSSの可能性がないか（dangerouslySetInnerHTML等）
- [ ] 外部入力のサニタイズ

#### G. itch.io互換性
- [ ] `vite.config.js` で `base: './'` が設定されているか
- [ ] 全てのアセットパスが相対パスか
- [ ] iframeで動作する前提の設計か
- [ ] ビルド後のdist/内で動作するか

### 3. コード品質チェック

- [ ] 1ファイルが500行を超えていないか
- [ ] ゲームロジックがUI層から分離されているか
- [ ] 定数がconfig.jsに集約されているか
- [ ] 未使用のimport/変数がないか
- [ ] console.logのデバッグ出力が残っていないか

### 4. バグレポート作成

発見した問題は以下の形式で報告してください：

```markdown
## Bug Report

### BUG-001: [タイトル]
- **深刻度**: Critical / High / Medium / Low
- **カテゴリ**: logic / ui / performance / i18n / ux / security / compat
- **影響ファイル**: `src/xxx.js`
- **問題**:
  [問題の説明]
- **再現手順**:
  1. xxx
  2. xxx
- **期待される動作**: xxx
- **実際の動作**: xxx
- **修正案**:
  [可能であれば修正方法を提案]
```
